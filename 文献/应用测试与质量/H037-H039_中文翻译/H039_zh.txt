arXiv:2509.04908v1 [cs.AI] 5 Sep 2025

摘要 现有用于 GUI 感知的多模态大语言模型（MLLMs）已经取得了很大进展。然而，先前方法中仍然存在以下挑战：1）它们基于文本自回归机制对离散坐标进行建模，这导致定位准确率较低且推理速度较慢。2）它们只能定位预定义的元素集合，无法解析整个界面，这阻碍了其广泛应用以及对下游任务的支持。为解决上述问题，我们提出 SparkUI-Parser，这是一种新颖的端到端框架，可同时实现更高的定位精度和对整个界面的细粒度解析能力。具体而言，我们并非采用基于概率的离散建模，而是在预训练的多模态大语言模型（MLLM）基础上，结合额外的 token 路由器和坐标解码器，对坐标进行连续建模。这有效缓解了 MLLMs 的离散输出特性和逐 token 生成过程固有的局限性，从而同时提升准确率和推理速度。为进一步增强鲁棒性，我们引入了一种基于改进匈牙利匹配算法的拒绝机制，使模型能够识别并拒绝不存在的元素，从而减少误报。此外，我们提出 ScreenParse，这是一个经过严格构建的基准，用于系统评估 GUI 模型在多样化场景中的结构化感知能力。大量实验表明，我们的方法在 ScreenSpot、ScreenSpot-v2、CAGUI-Grounding 和 ScreenParse 基准上始终优于 SOTA 方法。相关资源可在 https://github.com/antgroup/SparkUI-Parser 获取。

数据是自然图像，它们在 GUI 图像上的感知能力不足，这限制了它们在 GUI 特定语境中的有效性。自然图像通常包含具有多样对象和背景的复杂场景，而 GUI 图像则更具结构性，包含文本、按钮和输入框等具有特定功能和布局的元素。对于 GUI 场景感知和高级任务，理想的 GUI MLLMs 不仅需要理解各种设备上不断变化且信息密度高的界面，还需要准确执行基本操作，例如理解界面元素的语义并输出精确坐标。为构建这种强大的 GUI MLLMs，已有若干工作（Cheng et al. 2024; Xu et al. 2024; Gou et al. 2024）在 GUI 界面和任务上对 MLLMs 进行了后训练。它们专注于通过定位 GUI 界面上的图标或文本来提升 GUI 感知，并已展现出有前景的结果。然而，在感知 GUI 界面方面，这些先前方法仍然存在一些局限。首先，它们直接在通用 MLLMs 上进行微调，其中基于离散 token 概率分布的文本自回归机制会导致结果不准确且效率较低。因此，在定位任务中，它可能无法达到传统模型的准确率。此外，当需要快速响应时间时，高精度坐标的逐 token 预测方式可能成为瓶颈。其次，它们只关注定位指定元素并与之交互，无法对整个用户界面上更广泛的上下文以及元素之间的关系提供详细的解析式感知。第三，当需要定位不存在的元素时，先前模型会返回错误位置或生成无关响应，从而导致下游任务失败，这会显著影响用户体验和系统可靠性。为解决上述局限，我们提出一种新颖的端到端模型，以在一个模型内实现鲁棒定位和解析。具体而言，我们设计了一个“先路由后预测”的框架，用于高效处理视觉和语言信息，该框架由一个 MLLM、一个 token 路由器、一个视觉适配器、一个坐标解码器以及一个仅用于训练阶段的额外元素匹配器组成。来自 MLLM 的输出 token 由 token 路由器分类为文本 token 和视觉定位 token。文本 token 被解码为元素语义，而视觉定位 token 结合

• 我们还引入了用户界面上的多目标定位能力和对不存在元素的拒绝能力，从而增强真实世界场景中的鲁棒性和可靠性。

相关工作

通用 MLLM

近年来，通用 MLLM 取得了快速进展。GPT-4V（Yang et al. 2023）和 Gemini（Team et al. 2023）通过大规模网络级数据以及基于人类反馈的强化学习提升了性能，实现了强大的零样本视觉理解和推理能力。随后，LLaVA（Liu et al. 2023）、QwenVL（Bai et al. 2023）和 InternVL（Chen et al. 2024b）等开源模型利用精选数据集上的指令微调，高效地将 LLM 与视觉输入对齐。它们以端到端方式融合视觉编码器和语言解码器，支持包括音频和视频在内的多种模态。尽管取得了这些进展，通用 MLLM 主要是在自然图像而非 GUI 图像上训练的，因此在 GUI 特定场景中表现出有限的感知能力。

方法

当前基于 MLLMs 的 GUI 感知方法通常以离散文本 token 的形式生成坐标。这种基于词表映射的离散表示在一定程度上可能导致 grounding 不准确，从而与基于连续空间坐标回归的传统视觉 grounding 方法相比存在性能差距。为缩小这一差距并提升准确性，我们设计了一种高效的先路由后预测（route-then-predict）框架，该框架将语义理解与坐标优化解耦，同时利用 MLLMs 原生的空间感知能力，而不注入额外的感知模块。SparkUI-Parser 的整体架构如图 2 所示。

3.1

模型架构

该框架由一个 MLLM、一个 token 路由器、一个视觉适配器、一个坐标解码器和一个元素匹配器组成。给定一条用户指令和一个用户界面，token 路由器将 MLLM 的输出 token 分类为文本 token 和特殊 token（包括 [VG] token 和 [REJ] token）。随后，来自 [VG] token 和视觉适配器的多模态特征被输入坐标解码器，以生成精确坐标，而 [REJ] token 则被直接丢弃。文本 token 默认使用 MLLM 进行解码。在训练阶段，采用元素匹配器来建立预测元素与真实标注元素之间的语义和空间对应关系，从而有效缓解输出顺序变化的影响。接下来，我们对这一过程进行详细阐述。MLLM 尽管预训练 MLLM 具有良好的多模态理解能力，但其缺乏理解和定位 GUI 界面的能力。为了增强 MLLM 的 GUI 感知能力，我们使用 LoRA（Hu et al. 2022）对预训练 MLLM 的视觉部分和语言部分进行微调。用户指令 Tinstruction 和界面 I 首先由微调后的 MLLM FMLLM 处理，以获得语义特征 ftoken，如下所示：ftoken = FMLLM (I, Tinstruction )。（1）Token 路由器 为了基于离散 token 对连续坐标建模，我们通过 token 路由器将 MLLM 生成的输出 token 划分为语义 token 和位置 token。基于输出 token 的 logits，我们识别位置 token，包括 [VG] token 和 [REJ] token。[VG] token 指代目标的边界框，并且

4.3

评估指标

由于这是首次对用户界面的解析结果进行评估，我们提出从两个方面对模型进行量化：边界框性能和语义性能。边界框性能 引入了两个指标来评估边界框性能：元素召回率和元素精确率，它们为模型在 GUI 解析任务中的性能提供了全面视角。元素召回率定义为正确定位的元素数量与真实标注元素总数之比，而元素精确率定义为正确定位的元素数量与预测元素总数之比。较高的元素召回率表明模型能够有效识别元素的位置，确保很少遗漏实际元素。较高的精确率表明模型定位出的多数元素确实存在于用户界面中，从而最小化错误定位的数量。语义性能 引入语义相似度来评估语义性能，其通过计算模型预测语义与匹配边界框的真实标注之间的一致性来实现，涵盖文本相似度和图标描述相似度。语义相似度评估模型为已定位元素提供语义准确描述的能力，这对于需要深度 GUI 理解的任务至关重要，例如智能用户交互和上下文感知辅助。

5 5.1

实验

实验设置

网络架构 在我们的工作中，我们使用 InternVL2.5-8B（Chen et al. 2024a）作为基础 MLLM FMLLM，并且视觉适配器 Fadapter 是一个通道数为 [4096, 2048, 1024, 256] 的 MLP 模块。坐标解码器是一个基于 transformer 的架构，采用交叉注意力机制，其中隐藏状态维度为 256，并最终通过解码头回归到坐标。实现细节 我们采用 8 块 NVIDIA 80G A100 来训练整个框架，并且在训练期间解冻视觉编码器。LoRA（Hu et al. 2022）被用于微调 MLLM 中的视觉和语言部分，其中 lora-rank r 和 lora-alpha α 分别设置为 16 和 32。我们还采用 CosineLRScheduler 作为学习率调度器，其中学习率和 warmup 比例分别设置为 3e−5 和 0.03。在训练过程中，式 5 中的 λCE、λ1 和 λIoU 分别设置为 2.0、4.0 和 1.0，而元素匹配器中式 4 的 λIoU 和 λsem 均设置为 1.0。数据集 我们的训练数据由开源英文数据集和自标注中文数据集组成，包括移动端、桌面端和 Web 领域的 515k 个训练样本。对于英文数据集，我们为定位任务采样 400k 个训练样本，并额外标注 5k 个解析样本。对于中文数据集，我们在自收集的用户界面上标注了 100k 个定位样本和 10k 个解析样本。（更多数据集细节见补充材料。）此外，为了优化训练效率并增强模型鲁棒性，我们实现了一种多目标指令范式，其中每个训练实例随机包含多个目标，不同于传统的单目标方法。基线 我们将 SparkUI-Parser 与各种基线进行比较，包括闭源商业模型 GPT4o（Hurst et al. 2024）、Claude Computer Use（Anthropic 2024）、Gemini 2.0（Project Mariner）（GoogleDeepmind 2024），以及开源基础模型 Qwen2.5-VL 系列（Bai et al. 2025）、InternVL2.5 系列（Chen et al. 2024a）和学术 GUI 模型 SeeClick、CogAgent（Hong et al. 2024）、Aguvis（Xu et al. 2024）、OS-Atlas（Wu et al. 2024）、UGround（Gou et al. 2024）和 UI-TARS（Qin et al. 2025）。

解析结果

为了评估模型对用户界面元素语义和坐标的全局感知能力，我们使用所提出的基准 ScreenParse，该基准包含英文和中文用户界面。预测元素的平均耗时是在 NVIDIA 80G A100 上通过 transformer 框架获得的。如表 3 所示，SparkUI-Parser 在解析各种用户界面方面表现出优越性能。商业模型以及大规模模型只能定位少量元素，显示出较差的召回率和精确率。同时，它们所定位元素的语义相似度较低，表现出对元素语义理解的偏差。通过解耦解析任务中的语义和坐标，并在训练过程中使用元素匹配器来保持正确的元素优化，我们的模型相较其他模型在 ScreenParse 上取得了显著提升，同时也显著降低了单个元素的平均推理时间。

6

结论

本文提出了一种“先路由后预测”的框架，用于解耦用户界面元素的语义感知与位置感知。通过利用专门的坐标解码器将位置特征映射到连续坐标空间，而不是通过传统文本自回归方法生成坐标 token，我们的方法在以下基准上取得了

最先进的性能，同时具有最高的推理效率：ScreenSpot、ScreenSpot-v2、CAGUI-Grounding 和 ScreenParse。与此同时，我们在 GUI 场景中引入了拒绝机制，以避免模型生成错误的幻觉结果，并增强鲁棒性和可靠性。此外，我们发布了一个用户界面解析基准，用于评估模型对用户界面全局语义和位置的感知能力，以促进社区的进一步研究。
