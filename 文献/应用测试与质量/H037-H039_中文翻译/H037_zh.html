摘要 由多模态大语言模型（MLLMs）驱动的图形用户界面（GUI）智能体在理解和执行用户指令方面已经展现出令人印象深刻的能力。然而，从充斥着无关 UI 组件的高分辨率截图中准确定位与指令相关的元素，对于现有方法而言仍然具有挑战性。受到人类如何动态调整其感知范围以在复杂屏幕上定位任务相关区域的启发，我们提出 DRS-GUI，这是一个用于 GUI 定位的免训练动态区域搜索框架，可以无缝集成到现有 MLLMs 中。DRS-GUI 引入了一个轻量级 UI Perceptor，它执行三种类人的感知动作（Focus、Shift 和 Scatter），以逐步探索界面并生成区域候选。为了动态调度这些动作，我们进一步设计了一个基于蒙特卡洛树搜索（Monte Carlo Tree Search，MCTS）的 Action Planner。采用区域质量奖励来评估并选择高度与指令相关的区域，从而高效剪除冗余 UI 元素。实验表明，对于通用和 GUI 专用 MLLMs（Qwen2.5-VL-7B 和 UGround-V1-7B），DRS-GUI 在 ScreenSpot-Pro 上带来了 14% 的提升，显著增强了定位性能和泛化能力。

视觉杂乱，也源于在交互过程中缺乏一种用于自适应调整感知范围的显式机制。现有 GUI 定位方法通常可归为两种范式，但二者都缺乏真正自适应的感知行为。第一种范式将定位表述为单步全屏预测，直接在整张截图上回归一个点或边界框 [9, 17, 41]。由于缺乏对注意力的显式控制，此类模型常常无法从背景杂乱中分离出与指令相关的线索。第二种范式采用多步细化策略，例如迭代裁剪或缩放 [16, 23, 36]，以逐渐将视图缩小到目标位置。尽管这些方法提供了渐进式聚焦，但其细化过程通常是仅前向的，并且缺乏机制来评估每一步是否改进了与指令的对齐，或在早期决策偏离正确轨迹时进行回溯 [39]。如图 1（a）所示，当轨迹偏离语义线索时，最终区域可能无法包含目标，从而使错误不可恢复。相比之下，人类行为遵循一种更灵活的模式：并非停留在单一路径上不断缩小范围，人类会

人类会扫描布局，逐步细化注意力，并在出现不确定性时后退或将注意力转移到替代区域。这种自适应且可逆的搜索过程自然地评估注意力是在朝向还是真正目标远离移动 [31, 37, 38]，而这是当前 GUI 定位流程所缺乏的能力。这些观察表明，一个可靠的 GUI 定位框架应具备两个关键能力：（1）动态感知，即当当前证据较弱时，能够修正或转移查看区域，而不是致力于单一的、不可逆的逐步缩小路径；以及（2）区域质量评估，即每个感知步骤都提供一个可测量的信号，用于指示搜索是否正在更接近目标，从而帮助防止早期错误不断累积。为实现这些能力，我们提出 DRS-GUI，这是一个免训练框架，引入动态区域搜索和决策过程来指导定位流程。DRS-GUI 通过提出候选区域、借助语义和结构线索评估其与指令的相关性，并在必要时选择细化或重定向动作，动态调整感知范围。这种自适应感知过程呼应了人类在视觉密集界面中选择性探索并调整注意力的方式。为支持这一过程，DRS-GUI 纳入了一个辅助 UI Perceptor，它建立在 UI 元素解析器 [21] 和文本嵌入模块 [29] 之上。Perceptor 提供结构化 UI 表征，有助于确定哪些区域携带与指令相关的线索。在这些线索的引导下，系统执行三种感知动作：Focus（收缩视图以获得更精细的细节）、Shift（将注意力重新定位到替代区域）和 Scatter（向外扩展以恢复更广的上下文）。为了动态调度这些动作，DRS-GUI 采用了一个 Action Planner，该规划器集成蒙特卡罗树搜索（Monte Carlo Tree Search, MCTS）[28]，以实现类人的视野调整规划。我们进一步设计了一种奖励，用于评估区域质量并确定候选区域是否应被进一步探索或被选为最终结果。这种逐步评估缓解了早期错误的累积，解决了纯前向细化所固有的脆弱性。DRS-GUI 不需要额外的模型训练或微调，使其成为现有多模态大语言模型（Multimodal Large Language Models, MLLMs）[4, 7] 的即插即用增强组件。通过动态生成和评估感知区域，DRS-GUI 减少了冗余的视觉杂乱，并提升了在高分辨率、元素密集界面上的定位鲁棒性。总之，我们的贡献如下：• 我们提出 DRS-GUI，这是一种免训练的 GUI 定位方法，可执行动态区域搜索，有效增强 MLLMs 的定位性能。• 我们引入了一种类人的感知–动作过程，以获得可靠的目标区域：UI Perceptor 提供与指令相关的线索来驱动三种感知动作，

而基于 MCTS 的 Action Planner 结合区域质量奖励来调控这些动作，以达成稳定的逐步决策。• 实验表明，DRS-GUI 在高分辨率 ScreenSpot-Pro 基准上取得了显著提升，突显了其鲁棒性和通用性。

2. 相关工作 GUI 智能体。随着强大的多模态大语言模型（MLLMs）[2, 7, 32, 48] 的出现，GUI 智能体自动化复杂任务 [10, 33, 44] 的能力迅速提升。这一能力的基石是定位 [41, 52]。当前方法主要分为两种范式。第一种是静态全屏范式，其中 SeeClick [9]、OSAtlas [42] 和 UGround [11] 等代表性系统构建大规模标注数据集，以微调 MLLMs 进行文本到坐标预测。尽管该静态范式在结构化布局中有效，但在高分辨率、元素密集的界面上往往会退化，因为视觉杂乱会分散注意力并导致定位不稳定。静态模型的局限性，加上近期“think-with-image”策略 [6, 53, 54] 的兴起，推动了第二种基于迭代细化的范式。这些方法采用多步放大过程：ECP [16] 和 R-VLM [23] 首先定位候选区域，再进行高分辨率细化；DiMo-GUI [39] 通过模态分离和迭代区域放大执行免训练 GUI 定位；LASER [36] 是一个自演化框架，使 VLMs 具备多步感知和自适应区域推理能力，以实现精确的指令相关坐标预测。然而，这些放大方法本质上是不可逆的：一旦正确区域在早期被丢弃，搜索便无法恢复，错误也会累积。相比之下，我们的方法采用一种动态感知规划器，可根据需要重定向并重新访问区域，从而在密集界面中实现更可靠的定位。视觉搜索。早期视觉搜索模型受人类眼动启发，将显著性先验与概率搜索或学习策略相结合，以顺序选择注视位置 [26, 31, 47, 49, 51]，但它们通常使用固定大小的注意力窗口，并侧重于复现扫描路径，而不是在杂乱场景中准确定位小目标。较新的工作如 SEAL [40] 将定位模块和视觉记忆与大型多模态模型耦合，以引导注意力转向有前景的区域；ViGoRL [25] 将视觉搜索表述为序贯决策过程，并使用强化学习（RL）学习具备空间定位能力的注视策略；DyFo [18] 模拟类人的动态聚焦机制，使 MLLMs 能够放大到细粒度区域；FaST [30] 通过轻量级适配器控制视觉推理速度，以实现快–慢视觉搜索。尽管这些方法在自然图像场景中显示出潜力，

但它们并未充分考虑 GUI 布局的极端密度和结构异质性。相反，我们将 GUI 定位表述为一个视觉搜索问题，并将 DRS-GUI 设计为动态区域级规划器，在语言引导下自适应探索候选区域，以更可靠地定位细粒度 GUI 目标。

3. 方法论

我们将动态区域搜索视为 GUI 定位的预备阶段，目标是在基础多模态大语言模型（MLLM）执行最终坐标预测之前识别出一个可靠的推理区域。DRS-GUI 通过在语言引导下生成、评估并细化候选区域来调整视野，使得定位仅在到达语义上合适的界面区域之后才发生。如图 2 所示，DRS-GUI 包含两个协同工作的模块。UI 感知器（UI Perceptor）将当前区域解析为结构化 UI 元素，并提供与指令对齐的语义线索，同时还执行三种区域级动作（Focus、Shift、Scatter）以生成候选视图。随后，MCTS 动作规划器（MCTS Action Planner）调度这些动作以构建区域搜索树。在区域质量奖励的引导下，DRS-GUI 动态搜索并修正其感知范围，以在将其传递给基础 MLLM 进行最终坐标预测之前识别出最合适的区域。这种先搜索后预测的范式减少了视觉冗余并缓解了错误累积，从而在密集、高分辨率的 GUI 环境中实现更稳定且更可解释的定位。

3.1. 问题形式化

GUI 定位任务是将自然语言指令 T 映射到截图 Sfull ∈ RH×W ×3 中其对应 UI 元素的像素级坐标 p = (x, y)。然而，直接在 Sfull 上进行定位并不可靠，因为界面分辨率高，并且密集填充了视觉上相似但与指令无关的元素。因此，我们将定位重新表述为一个先搜索后预测的过程。区域搜索策略 πS 首先识别与指令相关的区域 Rbest ⊂ Sfull，而基础定位模型 M 仅在该区域内执行定位：

R_{\text {best}} = \pi _S(S_{\text {full}}, T), \qquad p = M(R_{\text {best}}, T) (1)

DRS-GUI 的目标是将 πS 实现为一种高效且可逆的动态搜索过程，从而在不进行模型再训练的情况下提高定位稳定性。

3.2. UI 感知器

UI 感知器为 DRS-GUI 提供感知基础。给定一个区域 R ⊂ Sfull，它将可见界面解析为一组结构化 UI 元素，并生成

具有 UI 领域感知能力的语义嵌入，用以引导感知动作的执行，从而优先考虑有意义且与用户指令在语义上对齐的区域。我们采用 OmniParser V2 [21] 来提取 UI 元素：

U = \{\, u_i \mid u_i = [\, b_i, d_i, i_i \,] \,\}_{i=1}^{N}

(2)

其中 bi 表示元素 i 的边界框，di 表示其语义描述（来自 OCR 识别文本或图标描述生成），ii ∈ {0, 1} 表示该元素是否可交互。为了建立更好的语义对齐，我们使用 instructor-large embedder [29] 为指令 T 和元素描述 di 生成领域特定嵌入。领域特定前缀 Pd 根据当前 GUI 截图的应用类型和系统类型进行设计，并在编码时前置，从而确保指令和 UI 元素都被嵌入到与当前活动 GUI 环境对齐的统一交互语境中。具体而言，我们计算

\begin {aligned} e_T &= \text {Embedder}(P_d \oplus T), \\ e_{d_i} &= \text {Embedder}(P_d \oplus d_i) \end {aligned} (3)

并对最终隐藏状态进行均值池化，以获得固定长度向量。共享的领域前缀将指令嵌入 eT 和元素嵌入 edi 置于一致的语义空间中。随后，为每个元素计算语义相关性得分 si = cosine(eT , edi )，这些得分作为感知动作执行和区域奖励评估的核心线索。利用相关性得分列表 {si}，UI 感知器支持三种类似人类的感知动作，用于在定位之前动态调整搜索区域的空间焦点：

Focus。Focus 将当前查看区域收缩到与指令高度相关的 UI 元素周围。在相关性得分 {si} 的引导下，感知器选择排名前 p% 的 UI 元素，并移除其框中心显著偏离簇质心的空间离群点。其余元素由最小外接边界框包围，以形成紧凑裁剪区域。如果相对于前一区域，该裁剪区域收缩不够充分，则迭代剪除最远的元素，直到满足目标收缩比例。这一聚类与剪枝步骤在抑制视觉杂乱的同时集中语义内容。

Shift。Shift 将搜索迁移到界面的不同区域。当高相关性线索出现在与当前视图在空间上分离的区域中时，Shift 会围绕这些线索重新居中该区域。为保持稳定性，这些元素会根据一致的布局方向（例如，上方区域、左侧区域）进行分组，并且构建所得区域时尽量与前一个区域保持最小重叠。因此，Shift 执行的是一种有目的的注意力迁移——允许搜索移出信息不足的区域。

3.3. 蒙特卡洛树搜索动作规划器 MCTS Action Planner 作为决策模块，用于调度感知动作并在区域上构建可逆的搜索轨迹。我们将每个区域状态 S = (R, U, {si }) 视为搜索树中的一个节点，其中 R 是当前区域裁剪，U 是解析得到的 UI 元素集合，{si } 表示由 UI Perceptor 计算的元素级语义相关性分数。感知动作 a 的空间定义为 A = {Focus, Shift, Scatter}，其中每个动作都会直接将当前区域变换为一个新的候选区域，从而形成感知树的边。在固定搜索预算下，规划器迭代执行以下四个阶段：节点选择。搜索从根节点开始，根节点通过对完整图像执行一步 Focus 初始化，以获得粗略的全局视图，确保后续 Scatter 和 Shift 动作在有意义的区域上运行。在每次迭代中，规划器根据应用于树的置信上界（Upper Confidence bounds applied to Trees, UCT）策略 [15] 选择子节点来遍历感知树，该策略在利用高奖励区域和探索访问较少的备选区域之间取得平衡。

a^\ast = \arg \max _{a \in A(S)} \left [ Q(S, a) + c \sqrt {\frac {\ln V(S)}{V(S, a)}} \right ]

(4)

区域质量奖励的影响 接下来，我们分析不同的区域质量奖励如何影响定位准确率。如表 5 所示，仅引入交互式加权相关性（Interactive Weighted Relevance）项即可带来 2.0% 的增益，突显了其在引导注意力转向语义上有意义且可操作区域方面的重要性。加入 UI 覆盖一致性（UI Coverage Consistency）奖励通过鼓励空间完整性带来了轻微的额外提升，尽管过度扩张有时会引入冗余，并在 Web 布局上造成小幅下降。纳入语义集中度（Semantic Concentration）奖励后，取得了 91.8% 的最佳整体准确率，并显著提升了在复杂 Web 场景上的表现。这表明，促进语义紧凑性有助于模型保持关注和稳定性，并与相关性和覆盖性的作用互补，从而实现均衡的区域评估。感知迭代预算 为了研究迭代预算的影响，我们在将最大树深度固定为 3 的情况下改变 N。如图 8 所示，随着 N 增加，准确率持续提升，但当迭代次数超过 N = 8 后，性能增益变得很小，表明额外探索会引入冗余的区域评估，从而产生收益递减。因此，我们选择 N = 8 作为默认迭代预算，因为它在准确率和计算效率之间取得了较强的平衡。

5. 结论 我们提出了 DRS-GUI，这是一个免训练框架，通过在坐标预测之前插入动态区域搜索阶段来改进 GUI 定位。轻量级 UI 感知器（UI Perceptor）和三种感知动作在基于 MCTS 的规划器和区域质量奖励的引导下，有助于将“看哪里”与“预测什么”解耦，并聚焦于紧凑且与指令相关的区域。在 ScreenSpot、ScreenSpot-V2 和 ScreenSpot-Pro 上的实验表明，相较于强大的通用 MLLM 和 GUI 专用 MLLM，该方法实现了持续提升，尤其是在高分辨率、元素密集的界面上。
